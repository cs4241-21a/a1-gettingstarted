Create JS objects:

Command to create object

Object.create()

Generally creating a object

    myPrototype = {
        test: function() {
            console.log( "This is a test")
        }
    }

Run function:

myPrototype.text()
response: This is a test

Delegator

delegator = Object.create(myPrototype)

Has access to prototype object methods

delegator.text()
response: This is a test

Change object's prototpye

delegator.__proto__ = { test3:1 }

Define a function

        double = function(number) {
            return number * 2
        }

        let n = 3
        triple = function() {
            n = n * 3
            return n
        }

The triple function has a side effect of modifying variables outside the scope of the function.
JS isn't really a function programming language because it allows you to have side effects

Closures

Enable a function to have access to data inside the scope of another function (building chains of functions)

        closure = function() {
        let z = 0
        return function() { return z }
        }

        myclosure = closure()

        myclosure()
        response: 0

Function can have properties

myclosure.text = 3

They basically act as objects
